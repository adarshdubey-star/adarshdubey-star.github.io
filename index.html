How Poke Thinks — AI + Quality

Two models working together, plus smart deduplication

Two-Model Strategy

Claude Sonnet 4.6 Review — generates findings. Each file gets:

  • The diff + complete file at HEAD
  • Blast radius from AST code graph
  • Shared codebase context (API endpoints, Kafka topics, tables, flags, auth)
  • Our team rules + PR description

Confidence scoring: 1-10 per finding. Below 6 = auto-dropped.

Gemini 2.5 Pro Critique — validates every finding:

"Is this real, or did code just move? Does endpoint cardinality contradict the assumption?"

Only findings that pass survive. This is where false positives die.

Both models free via Red Hat models.corp / GitHub Models — zero cost.

Smart Deduplication

14 files with the same concern? You get 1 comment, not 14.

1. Concept grouping

Finds domain patterns: "org_id" + "cross-tenant" → same concept. 14 findings → 1 listing all affected files.

2. Keyword similarity

Findings sharing >40% of meaningful words get merged.

3. Per-lens cap

Max 5 per lens, highest severity first.

Result: focused reviews — not walls of noise.
